# Comms Intake

## Phases

### 1. Intake
- Read meeting notes from `docs/client-communication/`
- Extract: client requests, technical requirements, actionable items, deferred items
- Categorize: Quick Win (<2h) | Feature (2-4h) | Epic (4h+)

### 2. Status Evaluation
For each item:
- Search codebase for existing implementation
- Status: ✅ Done | ⚠️ Partial | ❌ Not Started
- Generate status report: `docs/client-communication/{date}-status-autogenerated.md`

### 3. Planning
- Create implementation plan artifact
- Update `roadmap.yml` with new items (use `RM-{version}-###` IDs, include `prd_refs`)
- Create sprint files in `.seedgpt/sprints/` (use `SP-{sprint}-###` story IDs, include `prd_refs` and `roadmap_refs`)

### 4. User Review
- Request approval before execution

### 5. Execution
- Execute using `_tdd-loop.md`
- Update task.md progress
- Commit after each sprint

### 6. Completion
- Mark sprints complete
- Update roadmap status
- Notify user

## File Locations
| Artifact | Location |
|----------|----------|
| Meeting Notes | `docs/client-communication/` |
| Roadmap | `.seedgpt/roadmap.yml` |
| PRD | `.seedgpt/PRD.yml` |
| Sprints | `.seedgpt/sprints/` |

See: `_contract.md`, `sprint-plan.md`
